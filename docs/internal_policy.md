## 1. Общие положения

1.1. Настоящая Политика в отношении обработки персональных данных (далее — Политика)
разработана в соответствии с требованиями Закона Республики Беларусь от 07.05.2021 № 99-З
«О защите персональных данных» (далее — Закон № 99-З).

1.2. Оператор персональных данных: самозанятая **[ФИО]**, УНП **[УНП]**,
зарегистрированная по адресу **[адрес регистрации в РБ]** (далее — Оператор).

1.3. Действие настоящей Политики распространяется на все процессы обработки персональных
данных (далее — ПДн), осуществляемые Оператором с использованием средств автоматизации и без них.

1.4. Целью настоящей Политики является обеспечение защиты прав и свобод субъектов
персональных данных при обработке их ПДн Оператором.

## 2. Перечень обрабатываемых ПДн

| Категория | Состав |
|---|---|
| Идентификационные | email, имя (при указании) |
| Контактные | телефон, Telegram username |
| Технические | IP-адрес, User-Agent, браузерный fingerprint, cookie |
| Бизнес-сведения | URL сайта, название бренда, описание ниши, регион |
| Метаданные согласий | timestamp проставления чекбокса, IP в момент согласия |

Специальные категории ПДн (раса, политические взгляды, здоровье, биометрия) **не обрабатываются**.

## 3. Цели обработки

3.1. Предоставление услуги бесплатного аудита AI Visibility субъекту ПДн.

3.2. Связь с субъектом по результатам аудита.

3.3. Отправка follow-up писем (только при наличии согласия, с возможностью отписки).

3.4. Агрегированная аналитика воронки конверсии.

3.5. Защита сервиса от ботов и злоупотреблений.

## 4. Правовые основания обработки

4.1. Основное правовое основание — **согласие субъекта** (ст. 6 Закона № 99-З).

4.2. Согласие выражается путём проставления отдельных чекбоксов на формах сайта.

4.3. Зафиксированы два раздельных согласия:
- общее на обработку ПДн;
- на трансграничную передачу ПДн в Российскую Федерацию и США.

4.4. Каждое согласие фиксируется в базе данных с timestamp и IP-адресом субъекта.

## 5. Перечень действий с ПДн

| Действие | Цель | Средство |

| Сбор | оказание услуги | формы на сайте |

| Обезличивание | завершение взаимодействия | автоматизированно по сроку |
| Удаление | требование субъекта | по запросу на privacy@catcore.ru |

## 6. Меры защиты

### 6.1. Организационные меры

- Назначен ответственный за защиту ПДн: **[ФИО ответственного, для самозанятой — она сама]**.
- Назначение оформлено приказом № **[номер приказа]** от **[дата]**.
- Доступ к ПДн имеют только лица, чьи функции этого требуют (см. раздел 7).
- Все лица, имеющие доступ, ознакомлены с настоящей Политикой под подпись.
- Регулярное обучение мерам защиты ПДн — не реже 1 раза в год.

### 6.2. Технические меры

- Передача данных только по HTTPS (TLS 1.2 и выше).
- Пароли хранятся в виде криптографических хэшей (bcrypt).
- База данных недоступна из публичного интернета (только через приватную сеть).
- Регулярное резервное копирование с шифрованием.
- Журналирование действий с ПДн (audit log).
- Защита от ботов: rate limiting, honeypot, Cloudflare Turnstile, browser fingerprinting.
- Защита от SQL-инъекций через ORM (SQLAlchemy).
- Защита от XSS через автоэскейп шаблонов (Next.js, Jinja2).

Все лица подписали обязательство о неразглашении ПДн.

## 8. Порядок реагирования на запросы субъектов

8.1. Все запросы от субъектов ПДн направляются на email **privacy@catcore.ru**.

8.2. Срок рассмотрения — **5 рабочих дней** с момента получения.

8.3. Ответ направляется на тот же email, с которого пришёл запрос.

8.4. В случае запроса об удалении ПДн — данные удаляются полностью, кроме записи согласия
(хранится 3 года для подтверждения правомерности).

## 9. Порядок реагирования на запросы НЦЗПД РБ

9.1. Все запросы от Национального центра защиты персональных данных Республики Беларусь
рассматриваются в приоритетном порядке.

9.2. Срок ответа определяется НЦЗПД, но не более установленного законодательством.

9.3. По первому запросу Оператор предоставляет:
- настоящую Политику;
- публичную политику с сайта;
- журнал согласий за запрошенный период;
- список обработчиков (третьих лиц);
- технические описания мер защиты.

## 10. Сроки хранения

| Тип данных | Срок |
|---|---|
| Активные ПДн (для оказания услуги) | 1 год с последнего взаимодействия |
| ПДн после прекращения взаимодействия | удаление в течение 30 дней |
| Записи согласий | 3 года после прекращения обработки |
| PDF-отчёты в S3 | 1 год |
| Системные логи | 90 дней |

## 11. Порядок уничтожения ПДн

11.1. По истечении срока хранения или по требованию субъекта — данные удаляются из всех
систем хранения (СУБД, S3, резервные копии в обозримом цикле ротации).

11.2. Процедура удаления автоматизирована — Celery Beat-задача проверяет даты последнего
взаимодействия и удаляет просроченные данные.

11.3. Факт удаления фиксируется в системе.

11.4. Резервные копии, содержащие ПДн, автоматически ротируются и не хранятся более срока,
указанного в разделе 10.

## 12. Трансграничная передача

12.1. Имеет место передача ПДн в:
- Российскую Федерацию (Timeweb, Yandex, Сбер, Bitrix24 в белорусской юрисдикции);
- Соединённые Штаты Америки (OpenAI, Google, DeepSeek, Perplexity).

12.2. Передача осуществляется на основании **отдельного явного согласия** субъекта.

12.3. LLM-провайдерам (Yandex, OpenAI, Google, DeepSeek, Perplexity, GigaChat) передаются
**только обезличенные нишевые промпты** — без email, имени, телефона.
